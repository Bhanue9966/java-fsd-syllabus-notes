### 1\. SQL Data Types (MySQL)

**Data Types** define the kind of data a column can hold (e.g., numbers, text, dates) and the amount of memory allocated for it. Choosing the correct data type is crucial for performance and data integrity.

| Category | Data Type | Description | Usage Example |
| :--- | :--- | :--- | :--- |
| **Numeric** | **INT** | A normal-sized integer (whole number). | `age INT` |
| | **DECIMAL (M, D)** | Exact numeric data type. `M` is total digits, `D` is digits after the decimal point. | `price DECIMAL(5, 2)` (e.g., 999.99) |
| | **FLOAT / DOUBLE** | Approximate numeric data types (for high-precision, non-critical calculations). | `latitude DOUBLE` |
| **String** | **CHAR (size)** | Fixed-length string. If data is shorter, it's padded with spaces. (Fast but less flexible). | `country_code CHAR(2)` |
| | **VARCHAR (size)** | Variable-length string. Saves space but requires 1-2 bytes overhead. (Most common). | `name VARCHAR(255)` |
| | **TEXT** | Stores large strings (up to 65,535 characters). | `product_description TEXT` |
| **Date/Time** | **DATE** | Stores the date only (YYYY-MM-DD). | `hire_date DATE` |
| | **TIME** | Stores the time only (HH:MM:SS). | `alarm_time TIME` |
| | **DATETIME** | Stores both date and time (YYYY-MM-DD HH:MM:SS). | `created_at DATETIME` |
| **Boolean** | **BOOLEAN** | A synonym for **TINYINT(1)**. Stores `0` (False) or `1` (True). | `is_active BOOLEAN` |

```sql
-- DDL Example: Using Data Types in a Table Creation
CREATE TABLE Products (
    product_id INT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    unit_price DECIMAL(6, 2),
    is_available BOOLEAN,
    last_updated DATETIME
);
```

### 2\. SQL Operators

**Operators** are reserved words or characters used primarily in the `WHERE` clause to filter, perform calculations, or combine conditions.

#### A. Arithmetic Operators üßÆ

Used to perform mathematical calculations on numeric columns.

| Operator | Description | Syntax Example |
| :--- | :--- | :--- |
| **`+`**, **`-`**, **`*`**, **`/`** | Addition, Subtraction, Multiplication, Division. | `SELECT salary * 1.10 AS new_salary` |
| **`%`** | Modulo (Returns the remainder of a division). | `SELECT id % 2` |

#### B. Comparison Operators üîç

Used to compare two expressions or values, primarily for filtering in the `WHERE` clause.

| Operator | Description | Syntax Example |
| :--- | :--- | :--- |
| **`=`** | Equal to | `WHERE status = 'Active'` |
| **`!=`** or **`<>`** | Not equal to | `WHERE city <> 'New York'` |
| **`>`**, **`<`**, **`>=`**, **`<=`** | Greater/Less than (or equal to). | `WHERE age >= 18` |

#### C. Logical Operators üß†

Used to combine multiple conditions or to negate a condition.

| Operator | Description | Syntax Example |
| :--- | :--- | :--- |
| **`AND`** | Returns TRUE if **all** conditions are TRUE. | `WHERE dept = 10 AND salary > 50000` |
| **`OR`** | Returns TRUE if **at least one** condition is TRUE. | `WHERE status = 'Pending' OR region = 'West'` |
| **`NOT`** | Inverts the boolean result of the expression. | `WHERE NOT (age < 18)` |

#### D. Special Operators ‚ú®

Used for specific filtering tasks that involve sets, ranges, or pattern matching.

| Operator | Description | Syntax Example |
| :--- | :--- | :--- |
| **`BETWEEN`** | Tests if a value is within a range (inclusive). | `WHERE quantity BETWEEN 10 AND 50` |
| **`IN`** | Tests if a value matches any value in a specified list. | `WHERE country IN ('USA', 'UK', 'CA')` |
| **`LIKE`** | Searches for a specified pattern in a column using wildcards (`%` and `_`). | `WHERE name LIKE 'A%s'` (starts with A, ends with s) |
| **`IS NULL`** | Tests specifically for NULL values (missing or unknown data). | `WHERE manager_id IS NULL` |

#### E. Bitwise Operators üßÆ (Advanced)

These operators perform operations on the individual bits of two integer expressions, bit by bit. They are typically used for flags or low-level binary manipulation.

| Operator | Description |
| :--- | :--- |
| **`&`** | Bitwise **AND** |
| **`|`** | Bitwise **OR** |
| **`^`** | Bitwise **XOR** (Exclusive OR) |
| **`~`** | Bitwise **NOT** (Inversion) |
| **`>>`** | Right Shift |
| **`<<`** | Left Shift |

```sql
-- Example: Using Bitwise AND to check if a flag (bit) is set
-- Assuming the 'permissions' column is an integer storing flag values
SELECT * FROM Users WHERE permissions & 4;
```

#### F. Compound Operators (Procedural/Shorthand) ‚ûï

In standard SQL (DDL/DML), compound operators typically refer to the combination of an operation and assignment. While not standard in DML `SELECT`/`WHERE` clauses, they are crucial in procedural contexts like **PL/SQL** or **T-SQL** for updating variables.

| Operator | Description | Equivalent | Context |
| :--- | :--- | :--- | :--- |
| **`+=`** | Compound Addition/Assignment | `variable = variable + expression` | PL/SQL Variables, Stored Procedures |
| **`-=`** | Compound Subtraction/Assignment | `variable = variable - expression` | PL/SQL Variables, Stored Procedures |
| **`*=`** | Compound Multiplication/Assignment | `variable = variable * expression` | PL/SQL Variables, Stored Procedures |

```sql
-- Example: Usage in a PL/SQL or Procedure context
-- DECLARE @counter INT = 10;
-- SET @counter += 5; -- The value of @counter is now 15

-- In standard SQL UPDATE, the operation is explicit:
UPDATE Accounts SET balance = balance + 100 WHERE account_id = 1;
```
