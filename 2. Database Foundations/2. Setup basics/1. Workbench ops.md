### 2\. Database (Schema) Operations (Cont.)

#### B. Selecting the Active Database

You must designate which database (schema) your subsequent queries should target using the `USE` **DDL** command.

| Method | Instructions | SQL Command Syntax |
| :--- | :--- | :--- |
| **GUI** | In the **Schema Navigator** (left panel), **double-click** the schema name (e.g., `my_project_db`). The name will be set as active and appear in **bold**. | \`\`\`sql
USE my\_project\_db;



#### C. Deleting (Dropping) a Database

**Warning:** This is a permanent **DDL** operation that deletes the schema and all associated tables, views, and data.

| Method | Instructions |
| :--- | :--- |
| **GUI** | Right-click the schema name in the Navigator and select **"Drop Schema..."** Confirm the permanent action. |

```sql
-- DDL Syntax to Delete a Database
DROP DATABASE my_project_db;
````

### 3\. Executing SQL Queries

The **SQL Editor** is the area where you write and submit code to the server for execution.

| Action | Instructions | Keyboard Shortcut |
| :--- | :--- | :--- |
| **Execute Single Statement** | Place your cursor within the statement and click the small **lightning bolt icon** next to the query. | **`Ctrl + Enter`** (Windows) / **`Cmd + Enter`** (Mac) |
| **Execute All** | Click the large **lightning bolt icon** in the main toolbar. | |
| **Viewing Output**| Results from `SELECT` queries (DQL) appear in the **Result Grid**. Status messages (rows affected, errors) appear in the **Action Output** panel. | |

```sql
-- Full Example Execution Sequence: Demonstrating DDL, DML, and DQL

-- 1. Create and activate a new database
CREATE DATABASE inventory_db;
USE inventory_db;

-- 2. Create a table (DDL)
CREATE TABLE Products (
    product_id INT PRIMARY KEY,
    name VARCHAR(100) NOT NULL
);

-- 3. Insert data (DML)
INSERT INTO Products (product_id, name) VALUES (1, 'Laptop');
INSERT INTO Products (product_id, name) VALUES (2, 'Monitor');

-- 4. Retrieve data (DQL)
SELECT * FROM Products;
```
