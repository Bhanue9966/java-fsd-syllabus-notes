**DDL** commands are used to define or modify the structure (schema) of the database and its objects (tables, indexes, views). DDL operations are automatically committed and cannot be reversed using `ROLLBACK`.

### 1\. CREATE Command

The `CREATE` command is used to build new database objects.

#### A. Creating a Database (Schema)

Creates a new, empty database.

```sql
-- Syntax for creating a database
CREATE DATABASE [IF NOT EXISTS] database_name;

-- Example
CREATE DATABASE ProjectManagementDB;
```

#### B. Creating a Table (Comprehensive Syntax)

Creates a new table, defining columns, data types, and setting up integrity rules using constraints.

| Constraint | Description |
| :--- | :--- |
| **NOT NULL** | Ensures a column must contain a value. |
| **UNIQUE** | Ensures all values in a column are distinct. |
| **PRIMARY KEY** | Uniquely identifies each record (combination of NOT NULL and UNIQUE). |
| **FOREIGN KEY** | Links a column to the PRIMARY KEY of another table, enforcing referential integrity. |
| **CHECK** | Ensures all values in a column satisfy a specific condition. |
| **DEFAULT** | Provides a default value for a column if none is specified during insertion. |


```sql
-- Example: Creating Department and Employee Tables with Constraints
CREATE TABLE Departments (
    dept_id INT PRIMARY KEY,
    dept_name VARCHAR(50) NOT NULL UNIQUE
);

CREATE TABLE Employees (
    emp_id INT PRIMARY KEY,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    salary DECIMAL(10, 2) CHECK (salary >= 40000), -- CHECK constraint
    hire_date DATE DEFAULT CURRENT_DATE(),          -- DEFAULT constraint
    dept_id INT,

    -- Foreign Key Definition
    CONSTRAINT fk_dept
    FOREIGN KEY (dept_id)
    REFERENCES Departments(dept_id)
);
```

### 2\. ALTER Command

The `ALTER` command is used to modify the structure of an existing table.

#### A. Adding a New Column

```sql
-- Syntax
ALTER TABLE table_name ADD column_name datatype [constraint];

-- Example: Add a 'phone_number' column
ALTER TABLE Employees
ADD phone_number VARCHAR(15) UNIQUE;
```

#### B. Modifying Column Data Type or Constraints

Used to change the definition of an existing column (e.g., changing VARCHAR size or adding NOT NULL).

```sql
-- Syntax (Using MODIFY COLUMN in MySQL/PostgreSQL)
ALTER TABLE table_name MODIFY COLUMN column_name new_datatype [new_constraint];

-- Example: Increase the size of the 'dept_name' column
ALTER TABLE Departments
MODIFY COLUMN dept_name VARCHAR(100) NOT NULL;
```

#### C. Renaming a Column

```sql
-- Syntax (MySQL/MariaDB uses CHANGE COLUMN)
ALTER TABLE table_name CHANGE COLUMN old_column_name new_column_name datatype [constraint];

-- Example: Rename 'dept_id' to 'department_id'
ALTER TABLE Employees
CHANGE COLUMN dept_id department_id INT;
```

#### D. Dropping a Column

```sql
-- Syntax
ALTER TABLE table_name DROP COLUMN column_name;

-- Example: Remove the 'phone_number' column
ALTER TABLE Employees
DROP COLUMN phone_number;
```

-----

### 3\. DROP Command

The `DROP` command is used to permanently delete an entire database object.

#### A. Dropping a Table

Deletes the table structure and all data within it.

```sql
-- Syntax
DROP TABLE [IF EXISTS] table_name;

-- Example
DROP TABLE IF EXISTS Employees;
```

#### B. Dropping a Database

Deletes the entire database, including all objects and data within it.

```sql
-- Syntax
DROP DATABASE [IF EXISTS] database_name;

-- Example
DROP DATABASE IF EXISTS ProjectManagementDB;
```

-----

### 4\. TRUNCATE Command

The `TRUNCATE` command quickly removes all records from a table by deallocating the pages used for the table data. It is a DDL command because it affects the table's structure/metadata (by resetting counters).

```sql
-- Syntax
TRUNCATE TABLE table_name;

-- Example
TRUNCATE TABLE TempData;
```

-----

### 5\. RENAME Command

The `RENAME` command is used to change the name of an existing table.

```sql
-- Syntax
RENAME TABLE old_table_name TO new_table_name;

-- Example: Rename a table
RENAME TABLE Tasks TO ProjectTasks;
```
