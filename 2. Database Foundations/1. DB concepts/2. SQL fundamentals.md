### 1\. Introduction to Database, SQL, and MySQL

| Term | Definition | Role |
| :--- | :--- | :--- |
| **Database** | An organized collection of structured data. | Stores the information (the tables, views, etc.). |
| **SQL** | **Structured Query Language**. The standard language used to interact with a relational database. | The tool/language used to manage and query the data. |
| **MySQL** | A popular **Relational Database Management System (RDBMS)**. | The software that physically hosts and runs the database. |

-----

### 2\. Rules for Writing SQL Queries

SQL is generally **case-insensitive** for commands, but following these rules improves readability and ensures consistency:

  * **Use Keywords in UPPERCASE:** Always write SQL keywords (e.g., `SELECT`, `FROM`, `WHERE`) in uppercase.
  * **Terminate Statements:** Every SQL statement must end with a **semicolon (`;`)**.
  * **String Values:** Enclose all string (text) values within **single quotes (`'`)**.
      * *Example:* `WHERE last_name = 'Smith'`
  * **Formatting:** Use indentation and line breaks for complex queries.

### 3\. Naming Conventions

Consistent naming is critical for maintainability:

| Element | Convention | Example |
| :--- | :--- | :--- |
| **Table Names** | **Plural nouns**, often **snake\_case**. | `customers`, `product_orders` |
| **Column Names** | **Singular nouns**, **snake\_case** (all lowercase with underscores). | `first_name`, `order_date`, `is_active` |
| **Primary Keys** | Usually `id` or `table_name_id`. | `customer_id` |
| **Foreign Keys** | Usually `referenced_table_id`. | `dept_id` (in the `employees` table) |

#### Comments in SQL Language

| Comment Type | Syntax | Use Case |
| :--- | :--- | :--- |
| **Single-Line Comment** | `-- [Your comment here]` | Explaining a specific line or temporary notes. |
| **Multi-Line/Block Comment** | `/* [Your comment here] */` | Commenting out large blocks of code or providing header documentation. |

-----

### 4\. SQL Command Types and Explanation (Overview)

SQL commands are grouped into five main categories based on their function.

| Command Type | Full Form | Purpose | Key Commands |
| :--- | :--- | :--- | :--- |
| **DDL** | **Data Definition Language** | Defines, changes, and removes the **structure** (schema) of the database. | `CREATE`, `ALTER`, `DROP`, `TRUNCATE`, `RENAME` |
| **DML** | **Data Manipulation Language** | Handles access to and manipulation of the **data** within the tables. | `SELECT`, `INSERT`, `UPDATE`, `DELETE` |
| **DCL** | **Data Control Language** | Manages user **access and permissions** to the database. | `GRANT`, `REVOKE` |
| **TCL** | **Transaction Control Language** | Manages **transactions** (units of work) to ensure data integrity. | `COMMIT`, `ROLLBACK`, `SAVEPOINT` |
| **DQL** | **Data Query Language** | Strictly retrieves data from the database. (Often considered a subset of DML). | `SELECT` ||
