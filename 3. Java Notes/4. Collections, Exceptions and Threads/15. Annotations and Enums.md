## 15\. Annotations and Enums

### 1\. Annotations (Metadata) üè∑Ô∏è

**Annotations** are markers or metadata that can be added to Java source code. They provide information about the code, but they are **not** part of the code itself and do not directly affect program execution.

  * **Purpose:** To provide information to the compiler, development tools (like IDEs), or runtime processing.

#### A. Predefined Annotations (Standard Markers)

Java provides several standard annotations used for basic code communication:

| Annotation | Location | Purpose | Example |
| :--- | :--- | :--- | :--- |
| **`@Override`** | Method | Indicates that a method is intended to override a superclass method. Compiler checks for correct signature. | `@Override public String toString() { ... }` |
| **`@Deprecated`** | Class, Method, Field | Marks an element as obsolete and discourages its use. Compiler issues a warning if used. | `@Deprecated public void oldMethod() { ... }` |
| **`@SuppressWarnings`** | Program element | Tells the compiler to ignore specific warnings (e.g., unchecked generics warnings). | `@SuppressWarnings("unchecked")` |
| **`@FunctionalInterface`**| Interface | Validates that an interface has exactly one abstract method. (Covered in 13) | `@FunctionalInterface public interface MyInterface { ... }` |

#### B. Custom Annotations

You can define your own annotations using the **`@interface`** keyword. This allows you to tag code for specific purposes, often used in frameworks (like Spring or JUnit).

```java
// 1. Defining the custom annotation
@interface MyCustomTag {
    // Annotation elements are defined like methods
    String author(); 
    int version() default 1; // Element with a default value
}

// 2. Applying the custom annotation
@MyCustomTag(author = "Alice", version = 2)
public class AnnotatedClass {
    // ...
}
```

### 2\. Enums (Enumerations) üî¢

An **Enum** (short for Enumeration) is a special data type that allows a variable to be a set of predefined constants. It is used when you need a fixed set of constants, such as compass directions, days of the week, or user roles.

  * **Implementation:** Enums are implicitly specialized **classes** that extend `java.lang.Enum`. Each constant defined in the enum is a public static final instance of that enum type.
  * **Safety:** Enums provide type safety. Unlike string or integer constants, the compiler only allows the predefined values to be assigned to an enum variable.

#### A. Basic Enum Definition

```java
public enum Day {
    // These are public static final constants
    SUNDAY, MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY
}
```

#### B. Enum Usage Example

```java
public class EnumDemo {
    public static void main(String[] args) {
        Day today = Day.WEDNESDAY;
        
        if (today == Day.SATURDAY || today == Day.SUNDAY) {
            System.out.println("It's the weekend!");
        } else {
            System.out.println("It's " + today.name()); // MONDAY, TUESDAY, etc.
        }
    }
}
```

#### C. Enums with Fields and Methods (Advanced)

Since enums are classes, you can define **fields (data)**, **constructors**, and **methods** for them.

  * **Rule:** Constructors in an enum must always be **private** (implicitly or explicitly).
  * **Purpose:** To associate specific data or behavior with each constant.

<!-- end list -->

```java
public enum TrafficLight {
    // 1. Define constants with associated data (duration)
    RED(30), 
    YELLOW(5), 
    GREEN(45); 

    // 2. Private field to store data
    private final int durationSeconds; 

    // 3. Private constructor to initialize the data
    private TrafficLight(int durationSeconds) {
        this.durationSeconds = durationSeconds;
    }

    // 4. Public method to access the data
    public int getDuration() {
        return durationSeconds;
    }
    
    // 5. Public method to define behavior
    public void displayAction() {
        if (this == RED) {
            System.out.println("STOP for " + durationSeconds + "s.");
        } else {
            System.out.println("PROCEED with caution.");
        }
    }
}
```
