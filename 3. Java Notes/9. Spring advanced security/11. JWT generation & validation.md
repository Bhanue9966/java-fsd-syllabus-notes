### 1\. JWT Implementation: Generation and Validation

To work with JWTs in Java, we need a specialized library. **JJWT (Java JWT)** is one of the most popular choices. The implementation requires two main components:

1.  **`JwtService`:** Handles the creation (signing) and validation (parsing) of the token using the server's secret.
2.  **`JwtAuthenticationFilter`:** Intercepts every incoming HTTP request to validate the token before the request hits the Controller.

### 2\. Token Generation (`JwtService`)

The generation process involves defining the claims and signing them with a secret key to produce the final JWT string.

#### A. Required Components

  * **Secret Key:** A base64 encoded string known only to the server.
  * **Expiration Time:** How long the token is valid (e.g., 15 minutes).

#### B. Generation Logic

```java
package com.app.security.jwt;

import io.jsonwebtoken.Jwts;
import io.jsonwebtoken.SignatureAlgorithm;
import org.springframework.security.core.userdetails.UserDetails;
// ... imports

@Service
public class JwtService {

    @Value("${jwt.secret}") // Loaded from application.properties
    private String SECRET; 
    
    @Value("${jwt.expiration}") // e.g., 900000ms (15 minutes)
    private long EXPIRATION_TIME; 

    // --- Generation Method ---
    public String generateToken(UserDetails userDetails) {
        
        // 1. Define Claims (payload)
        Map<String, Object> claims = new HashMap<>();
        claims.put("roles", userDetails.getAuthorities().stream()
                               .map(GrantedAuthority::getAuthority).collect(Collectors.toList()));

        return Jwts.builder()
            .setClaims(claims)
            .setSubject(userDetails.getUsername()) // 2. Subject is the username
            .setIssuedAt(new Date(System.currentTimeMillis()))
            .setExpiration(new Date(System.currentTimeMillis() + EXPIRATION_TIME)) // 3. Set expiration
            .signWith(SignatureAlgorithm.HS256, SECRET) // 4. Sign with secret
            .compact(); // 5. Produce the final string
    }
}
```

### 3\. Token Validation and Filtering

Validation ensures the token is genuine, not expired, and sets the user identity in Spring's Security Context. This happens inside a custom filter that we insert into the Spring Security chain.

#### A. The `JwtAuthenticationFilter`

This filter extends Spring's **`OncePerRequestFilter`** to run once per HTTP request.

```java
package com.app.security.filter;

// ... imports for Spring, JWT Service, UserDetailsService ...

@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {

    @Autowired
    private JwtService jwtService;
    
    @Autowired
    private UserDetailsService userDetailsService;

    @Override
    protected void doFilterInternal(
        HttpServletRequest request,
        HttpServletResponse response,
        FilterChain filterChain
    ) throws ServletException, IOException {
        
        final String authHeader = request.getHeader("Authorization");
        final String jwt;
        final String username;

        // 1. Check for token presence and "Bearer " prefix
        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            filterChain.doFilter(request, response);
            return;
        }

        jwt = authHeader.substring(7); // Remove "Bearer "
        username = jwtService.extractUsername(jwt); // Utility method to read 'sub' claim

        // 2. If token is valid and no user is currently authenticated
        if (username != null && SecurityContextHolder.getContext().getAuthentication() == null) {
            
            // 3. Load user details from the database
            UserDetails userDetails = this.userDetailsService.loadUserByUsername(username);
            
            // 4. Validate token (signature, expiration)
            if (jwtService.isTokenValid(jwt, userDetails)) {
                
                // 5. Create Authentication Token
                UsernamePasswordAuthenticationToken authToken = new UsernamePasswordAuthenticationToken(
                    userDetails,
                    null, // Credential is null for JWT
                    userDetails.getAuthorities()
                );
                
                // 6. Set the user in the Security Context
                SecurityContextHolder.getContext().setAuthentication(authToken);
            }
        }
        
        filterChain.doFilter(request, response); // Proceed to next filter/controller
    }
}
```

### 4\. Integrating the Filter into Security Configuration

Finally, we must tell Spring Security to place our custom filter **before** the standard filters that perform authorization checks.

```java
// Spring Security Configuration (SecurityFilterChain Bean)

@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
        // Disable CSRF for stateless APIs
        .csrf(AbstractHttpConfigurer::disable)
        
        // Ensure session management is STATELESS for JWT
        .sessionManagement(session -> session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
        
        // Add our custom filter before the standard authentication filter
        .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class)
        
        // Define authorization rules (e.g., allow /api/auth/** without token)
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/api/auth/**").permitAll()
            .anyRequest().authenticated()
        );

    return http.build();
}
```

By adding the filter, every protected request first passes through the `JwtAuthenticationFilter`, which establishes the user's identity before the authorization rules are applied.
