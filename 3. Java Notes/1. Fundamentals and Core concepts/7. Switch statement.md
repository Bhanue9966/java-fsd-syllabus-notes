## 7\. Switch Statement

The **`switch` statement** is a conditional control structure used when you need to choose one path from **many possible execution paths** based on the value of a single variable. It offers a cleaner, more readable alternative to a long `if-else if-else` ladder when checking for equality against several constant values.

### 1\. Basic Structure and Flow

The `switch` statement evaluates an expression (the **selector**), and then tries to match its value against the various `case` labels.

#### Syntax

```java
switch (expression) {
    case value1:
        // Code to execute if expression == value1
        break; // Stop execution and exit the switch block
    case value2:
        // Code to execute if expression == value2
        break;
    default:
        // Code to execute if no case matches (optional)
}
```

#### Key Components

  * **`expression` (Selector):** The variable whose value is being tested. In modern Java, this can be an `int`, `byte`, `short`, `char`, `String`, or `Enum`.
  * **`case`:** Defines a constant value to test against the expression.
  * **`break`:** **Crucial\!** This keyword terminates the `switch` block. Without it, execution will continue to the next `case` label, even if the case doesn't match the expression (this is called **fall-through**).
  * **`default`:** An optional block that runs if none of the `case` values match the expression.

### 2\. Example: Day of the Week

This example uses `break` to ensure only the intended code block runs.

```java
int day = 3;
String dayName;

switch (day) {
    case 1:
        dayName = "Monday";
        break;
    case 2:
        dayName = "Tuesday";
        break;
    case 3:
        dayName = "Wednesday";
        break; // Execution stops here!
    default:
        dayName = "Invalid Day";
}
System.out.println(dayName); // Output: Wednesday
```

### 3\. Understanding Fall-Through (The Need for `break`)

If you omit the `break` statement, the code will **fall through** and execute the code blocks for all subsequent `case` labels until a `break` or the end of the `switch` is reached.

This is rarely desirable, but it's useful for grouping cases that have the same outcome:

```java
char grade = 'A';

switch (grade) {
    case 'A':
    case 'B': // Grouping cases: both A and B fall through to the same output
        System.out.println("Good Job!");
        break;
    case 'C':
        System.out.println("Needs Improvement.");
        break;
    default:
        System.out.println("See the instructor.");
}
// Output: Good Job!
```

### 4\. Switch Expressions (Java 12+)

Modern Java introduced **`switch` expressions** (using the `->` arrow syntax) to simplify the structure and eliminate the need for `break` statements, automatically preventing fall-through.

The `switch` statement now returns a value.

#### Syntax (Switch Expression)

```java
String result = switch (expression) {
    case value1 -> valueIfTrue1;
    case value2 -> valueIfTrue2;
    default -> defaultValue;
};
```

#### Example (Using Switch Expression)

```java
int day = 3;

String dayType = switch (day) {
    case 1, 2, 3, 4, 5 -> "Weekday"; // Multiple cases handled with comma
    case 6, 7 -> "Weekend";
    default -> "Invalid";
};

System.out.println(dayType); // Output: Weekday
```
