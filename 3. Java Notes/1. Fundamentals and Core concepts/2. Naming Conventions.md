## 2. Naming Conventions

**Naming Conventions** are the non-mandatory, but universally adopted, rules and best practices for naming identifiers (like variables, methods, classes, etc.) in Java. Following these conventions makes your code:

* **Readable:** Other developers (and your future self!) can understand the code faster.
* **Maintainable:** Easier to modify and debug.
* **Professional:** It's the standard practice in the industry.

Java relies heavily on **Camel Case** for naming.

### üê™ The Four Main Conventions

| Identifier Type | Convention Used | Rule | Example |
| :--- | :--- | :--- | :--- |
| **Classes** and **Interfaces** | **PascalCase** (or Upper Camel Case) | Starts with an **uppercase** letter. If multiple words, each new word starts with an uppercase letter. | `EmployeeDetails`, `MyCalculator`, `Serializable` |
| **Methods** and **Variables** | **camelCase** (or Lower Camel Case) | Starts with a **lowercase** letter. If multiple words, the subsequent words start with an uppercase letter. | `calculateSalary()`, `firstName`, `currentBalance` |
| **Packages** | **lowercase** | All letters are in **lowercase**. Words are separated by dots (`.`). | `com.app.util`, `java.io`, `model.entities` |
| **Constants** | **UPPER_CASE** | All letters are **uppercase**. Words are separated by the **underscore** (`_`). Used for variables declared as `static final`. | `MAX_SIZE`, `PI_VALUE`, `DEFAULT_TIMEOUT` |

### üîç Specific Naming Rules and Guidelines

In addition to the conventions above, Java enforces strict rules:

| Rule Category | Description | Examples (Good vs. Bad) |
| :--- | :--- | :--- |
| **Start Character** | Identifiers must start with a **letter**, an **underscore** (`_`), or a **dollar sign** (`$`). | **Good:** `name`, `_age`, `$price` **Bad:** `1stName` |
| **Reserved Keywords** | You cannot use any of Java's **50 reserved keywords** (like `public`, `class`, `int`, `for`) as an identifier. | **Bad:** `int class = 10;` |
| **Case Sensitivity** | Java is **case-sensitive**. `myvariable` is different from `MyVariable`. | `int sum;` and `int Sum;` are two distinct variables. |
| **Descriptiveness** | Names should be meaningful and reflect the purpose of the variable or method. | **Good:** `isDataValid` **Bad:** `flg` (too short) |
