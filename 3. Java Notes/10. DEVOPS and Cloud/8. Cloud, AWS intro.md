### 1. What is Cloud Computing?

**Cloud Computing** is the on-demand delivery of IT resources (like computing power, database storage, applications, and other IT services) over the internet with **pay-as-you-go** pricing.

* **Core Principle:** It replaces the capital expense of buying and owning physical servers and data centers with the variable expense of renting resources only when you need them.

#### Key Benefits of Cloud Computing

| Benefit | Description |
| :--- | :--- |
| **Elasticity** | Automatically and dynamically scales capacity up or down in response to demand. |
| **Agility** | Rapidly provision new resources, significantly speeding up time-to-market. |
| **Pay-as-you-go** | You only pay for the resources you consume, often by the second or hour. |
| **Global Reach** | Deploy applications quickly in multiple geographical regions around the world. |

### 2. Cloud Service Models

Cloud services are categorized by who manages which part of the technology stack. The distinction between **IaaS**, **PaaS**, and **SaaS** is critical.

| Service Model | Full Form | What You Manage | What the Provider Manages |
| :--- | :--- | :--- | :--- |
| **SaaS** | Software as a Service | None (Just the data/user usage). | Everything (Application, Runtime, OS, Hardware). |
| **PaaS** | Platform as a Service | **Code/Application** and data. | Runtime, OS, Hardware, Networking. |
| **IaaS** | Infrastructure as a Service | **OS**, Middleware, Runtime, Application. | Servers, Storage, Networking, Virtualization. |

* **Analogy:** If software is a pizza: **IaaS** is buying the raw ingredients (managing everything). **PaaS** is ordering the pizza, which arrives pre-cooked on a delivery platform (managing your toppings, but not the oven). **SaaS** is eating the pizza at a restaurant (managing nothing).

### 3. Introduction to Amazon Web Services (AWS)

**AWS** is the worldâ€™s most comprehensive and broadly adopted cloud platform, offering over 200 fully featured services from data centers globally. It is an example of a **Public Cloud** deployment model, where services are offered to the general public over the internet.

#### AWS Global Infrastructure

AWS services run from physical data centers located in geographical regions. Understanding this hierarchy is key to designing resilient applications.

1.  **Region:** A physical location around the world where AWS clusters data centers (e.g., `us-east-1` in North Virginia, `eu-central-1` in Frankfurt). Applications should be deployed in a region closest to their target users for minimal latency.
2.  **Availability Zone (AZ):** One or more discrete data centers with redundant power, networking, and connectivity, separated from each other to prevent a single event (like a fire or flood) from affecting more than one AZ. A single AWS Region typically has 3 to 6 AZs.
3.  **Edge Locations:** Worldwide points used by services like **CloudFront (CDN)** to cache content and deliver it to end-users with the lowest possible latency.
