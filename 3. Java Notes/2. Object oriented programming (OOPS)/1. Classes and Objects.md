## 1\. Classes and Objects: Theory & Practical

OOP is a programming paradigm based on the concept of **objects**, which can contain data (fields/attributes) and code (methods/behaviors). The goal is to structure a program into simple, reusable blueprints.

### 1\. The Class: The Blueprint

A **Class** is a **template** or **blueprint** from which objects are created. It is a logical entity; it does not occupy any memory when defined.

  * **Definition:** A class is a collection of variables (attributes) and methods (behavior) that describe a common type of object.
  * **Syntax:**
    ```java
    public class ClassName {
        // Variables (data/attributes)
        dataType fieldName;
        
        // Methods (behavior/functions)
        returnType methodName() {
            // body of the method
        }
    }
    ```

### 2\. The Object: The Real-World Entity

An **Object** is a **real-world instance** of a class. It is a physical entity that occupies memory when created. Every object created from a class will have its own unique copy of the class's non-static variables.

  * **Creation:** Objects are created using the **`new`** keyword, which calls a constructor.
  * **Example: Class vs. Object**
    | Concept | Class (Blueprint) | Object (Instance) |
    | :--- | :--- | :--- |
    | **Example** | `Car` | `myCar` (a red Sedan), `yourCar` (a blue Truck) |
    | **Memory** | No memory allocated. | Memory allocated on the **Heap**. |

#### Object Creation Syntax:

1.  **Declaration:** `ClassName objectReference;` (Creates a reference variable on the Stack)
2.  **Instantiation:** `objectReference = new ClassName();` (Creates the actual object on the Heap)

<!-- end list -->

```java
public class Dog {
    String breed;
    int age; // Variables
    
    void bark() { // Method
        System.out.println("Woof!");
    }
}

// In the main method:
public static void main(String[] args) {
    // Creating an object (instantiation)
    Dog myDog = new Dog(); // myDog is the object reference
    
    // Accessing object's data and behavior
    myDog.breed = "Poodle";
    myDog.age = 3;
    myDog.bark(); // Output: Woof!
}
```

### 3\. Methods: Defining Behavior

A **Method** is a block of code that performs a specific action. It allows objects to communicate and modify their own state.

  * **Components:**
      * **Access Modifier** (`public`, `private`, etc.)
      * **Return Type** (`void`, `int`, `String`, etc.)
      * **Method Name** (camelCase)
      * **Parameters** (input values)
      * **Method Body** (the code)

#### Example of Methods

```java
// A method that accepts no input and returns nothing (void)
void greet() {
    System.out.println("Hello!");
}

// A method that accepts one input (int) and returns a value (int)
int add(int a, int b) {
    return a + b;
}
```

### 4\. Method Overloading

**Method Overloading** is a feature that allows a class to have **multiple methods with the same name**, but with different **parameter lists**.

  * **Key Rules for Overloading:** Methods must differ in:
    1.  **Number of parameters**
    2.  **Data type of parameters**
    3.  **Order of parameters**
  * **Note:** The return type or access modifier **alone** cannot differentiate overloaded methods.

<!-- end list -->

```java
public class Calculator {
    
    // 1. Overload: Two integer parameters
    int add(int a, int b) {
        return a + b;
    }
    
    // 2. Overload: Three integer parameters (Different number)
    int add(int a, int b, int c) {
        return a + b + c;
    }
    
    // 3. Overload: Two double parameters (Different type)
    double add(double a, double b) {
        return a + b;
    }
    
    // ERROR: Cannot overload simply by changing return type (compiler won't allow this)
    // double add(int a, int b) { return a + b; } 
}
```
